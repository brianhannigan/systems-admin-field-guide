# 01 - First 30 Days Survival Plan

## Purpose
Give a new sysadmin a structured way to survive the first month, rapidly understand the environment, reduce mistakes, and build credibility.

## Objectives
- Define what “good” looks like for this area
- Capture the minimum viable knowledge needed early
- Break the topic into trackable sub-sections
- Identify where labs, checklists, and scripts belong

## Core Questions to Answer
- What does this component cover operationally?
- What usually goes wrong here?
- What should be learned first vs later?
- What should be documented for repeatable team use?
- What can be automated?

## Outline
### A. Immediate Priorities\n- Learn the mission of the environment\n- Identify mission-critical systems\n- Determine top outage risks\n- Understand patching and maintenance rhythms\n\n### B. Questions to Ask\n- Which systems fail most often?\n- Which systems are highest priority?\n- What upgrades are already planned?\n- What does the team fear touching?\n\n### C. Systems to Map\n- Servers and roles\n- Network dependencies\n- Authentication paths\n- Monitoring and alerting\n- Backup and recovery flows\n\n### D. Documentation to Collect\n- Network diagrams\n- Asset inventory\n- Runbooks\n- Escalation contacts\n- Patch schedules\n- Vulnerability scan reports\n\n### E. Red Flags\n- Unknown service ownership\n- No rollback plans\n- Shared admin accounts\n- Untracked changes\n- Unsupported OS versions

## Suggested Deliverables
- First 30-day checklist\n- Discovery worksheet\n- Infrastructure mapping worksheet\n- Questions-to-ask interview sheet\n- Early warning / red flag reference
